大模型

  1. 指的是参数量极大（Billion+），模型本身显存和计算需求高
  2. 单卡放不下 / 单机训练时间太长
  3. 必须用分布式训练手段才能完成
  4. 典型例子：LLaMA3-8B、70B，GPT-4，Gemini 等

模型训练

包括正向传播、反向传播、梯度更新

  • 前向传播：根据当前参数计算模型输出和损失
  • 反向传播：用链式法则从损失往回，层层计算出各个参数的梯度
  • 参数更新：根据计算出的梯度，更新参数

分布式训练

  • 是为了解决大模型训练的计算 / 存储瓶颈
  • 拆分模型（TP）、拆分数据（DP）、拆分层（PP）、拆分状态（ZeRO / FSDP）
  • 需要分布式通信、同步、协调
  • 涉及并行策略 + 启动工具 + 通信后端 + 容错机制

分布式训练关键参数说明

  1. WORLD_SIZE
    总 GPU 数量，表示整个训练中参与的 GPU 数量。

  2. TP_SIZE（Tensor Parallel Size）
    Tensor 并行度：将单层模型的巨大参数拆分成多份，每个 GPU 计算一部分，多个 GPU 协作完成该层的前后向计算。

    • 减小单 GPU 显存压力
    • 支持更大模型

  3. PP_SIZE（Pipeline Parallel Size）
    Pipeline 并行度：将模型的不同层划分为多个“段”，每个 GPU 负责其中一段的前向和反向传播。

    • 进一步分摊显存
    • 支持更深更大的模型结构

  4. DP_SIZE（Data Parallel Size）
    数据并行度：经典的并行方式，每个 GPU 保持模型完整副本，处理不同的 batch 数据，并在每个 step 同步梯度。

    • DP_SIZE = WORLD_SIZE / (TP_SIZE × PP_SIZE)
    • 表示一个模型并行组的数量，每组对应一个数据并行副本

  5. MICRO_BATCH_SIZE
    微批大小：指单次前向 + 反向的 batch 大小。多个微批连续执行，累计梯度后再更新一次参数。

  6. NUM_MICROBATCHES
    一个 step 内，在梯度累计前连续运行的微批次数量。

  7. GLOBAL_BATCH_SIZE
    全局批大小：一次参数更新时模型“整体”看到的样本数，是训练稳定性和模型效果的重要因素。大模型常用大 batch，有助于加速收敛。

    • GLOBAL_BATCH_SIZE = DP_SIZE × NUM_MICROBATCHES × MICRO_BATCH_SIZE
    • 相当于小模型中一个 step 的 batch_size，只不过这里是分布式场景中整组模型并行副本累计处理的总样本数

torchrun 启动机制

用于启动多个分布式训练进程，配合 Megatron-LM 驱动大模型训练。

  • 自动为每个 GPU 启动一个 Python 进程（单机多卡）
  • 多节点时，协调每个节点与 master 建立通信
  • 初始化 torch.distributed 环境
  • 每个进程内使用 torch.distributed.get_rank() 获取全局 rank
  • 后续的 DDP、模型并行、AllReduce 等都依赖这个初始化

分布式常见参数说明

参数名 说明
world_size 总进程数 = nnodes × nproc_per_node
rank 每个进程的全局编号（0 ~ world_size-1）
local_rank 每个节点内的进程编号（0 ~ nproc_per_node-1）
master_addr & master_port 用于集群通信的协调节点

分布式通信机制

基本通信原语（PyTorch Distributed）

  • AllReduce：所有 GPU 的张量求和/平均 → 同步梯度
  • Broadcast：从 rank 0 广播数据到其他进程
  • AllGather / ReduceScatter：多张量收集与分发
  • Barrier：全体进程阻塞同步，常用于初始化或 barrier 控制点

通信算子与应用场景

通信算子 作用描述 应用场景
AllReduce 张量求和/平均，同步到所有 GPU 同步梯度（DDP）
Broadcast 从 rank 0 广播张量 初始化模型参数
AllGather 所有进程张量拼接 模型并行中拼接切片
ReduceScatter 先规约后切片 张量并行中中间通信
Barrier 阻塞等待所有进程 开始训练或加载 checkpoint

通信分组（以 Megatron-LM 为例）

  • 数据并行组：同步梯度
  • 张量并行组：分片权重计算中通信
  • 流水线并行组：跨 stage 的激活通信

通信后端

后端名称 说明
NCCL CUDA 最快，NVIDIA 优化，支持 NVLink / PCIe / IB
MCCL MUSA 的通信后端（国产 GPU）
Gloo 通用后端，适用于 CPU、调试

NCCL Ring AllReduce / AllGather / ReduceScatter 原理简述

  • AllReduce：全卡归约并广播，分布式梯度同步的核心算子。
  • Ring AllReduce：将计算/通信均匀分布到所有进程，无 master 瓶颈，通信效率高。
  • ReduceScatter：AllReduce 的前半部分，规约后切片分发。
  • AllGather：AllReduce 的后半部分，将切片拼接广播。
  • Broadcast / Barrier：点对多点广播，全体同步阻塞。

通信模块的四个层次

  1. 通信原语（逻辑层）：AllReduce、Broadcast 等
  2. 通信库（软件层）：NCCL、MPI、Gloo
  3. 协议与驱动（接口层）：RDMA、TCP、驱动程序
  4. 真实硬件（物理层）：GPU、PCIe、NVLink、IB、交换机

分布式并行策略

单机多卡策略

数据并行（Data Parallel）

  • 每卡持有模型完整副本，分批处理不同输入，梯度通过 AllReduce 同步
  • ✅ 简单易实现，支持成熟（如 PyTorch DDP）
  • ❌ 模型必须单卡可容纳；AllReduce 开销随卡数上升

张量并行（Tensor Parallel）

  • 将某一层的参数（如矩阵）按维度切分，分卡并行计算
  • 用于降低单卡显存
  • 通信较频繁，需要 AllGather、ReduceScatter
  • 应用：Megatron-LM 的 1D 分片

流水线并行（Pipeline Parallel）

  • 将模型划分成多个 Stage，多个微批次以流水线形式穿过各卡
  • 优点：大幅降低激活显存占用
  • 缺点：流水线填空/排空带来利用率下降，通信复杂

混合并行（Hybrid）

  • 常见为 2D 或 3D 并行：张量 + 流水线 + 数据并行组合
  • 框架支持：Megatron-LM、DeepSpeed（3D Parallel）

多机多卡策略

跨机数据并行

  • 每机内部数据并行，节点间通过 NCCL 做梯度 AllReduce
  • 关键：网络带宽、拓扑感知、NCCL 环境变量调优

跨机模型并行

  • 张量并行 / 流水线并行可跨机拓展
    • 跨机张量并行：tensor 切片分布于多机
    • 跨机流水线并行：stage 可跨机部署
    • 支持 3D / 4D 混合并行（如 DeepSpeed）

主流框架支持对比

技术 / 框架 支持的并行类型 说明
PyTorch DDP 数据并行 简单易用，入门推荐
Megatron‑LM 1D 张量 + 流水线 + 数据并行 超大模型训练常用
DeepSpeed ZeRO (1/2/3) + Tensor + Pipeline + Data 支持上千亿参数
FairScale FSDP + 张量并行 PyPI 轻量包，适合实验

从单机到多机训练的推荐步骤

  1. 单机开发调试

    • 先用数据并行，确保代码正确
    • 用小模型 + 小 batch 验证显存/性能

  2. 加入模型并行

    • 单卡显存不够 → 加入张量并行 / 流水线并行
    • 加入激活检查点，节省显存

  3. 扩展到多机

    • 配置 NCCL、InfiniBand、确认 GPU 拓扑
    • 设置 init_process_group()

  4. 性能调优

    • 通信：调 NCCL 环境变量
    • 流水线：设置 micro-batch 数量、stage 划分平衡通信
    • ZeRO：根据显存与带宽选 Stage1/2/3

  5. 监控与 Profiling

    • 显存：torch.cuda.memory_summary()
    • 时间：PyTorch Profiler、DeepSpeed Profiler
    • 通信：nvidia-smi topo --matrix

总结

类型 特点
数据并行 入门首选，易扩展，显存限制
模型并行 张量 + 流水线，缓解显存压力，复杂度提升
ZeRO 拆分优化器/梯度/参数，占用最低显存
混合并行 超大模型训练主流，3D 组合策略
